À propos de Devento

Pourquoi un énième site technique pour les dévs ?​

Des formats courts​

Le but de cette partie est de faire des formats courts.
Tu trouveras facilement énormément de sources diverses et variées contenant de longues explications très détaillées. D'ailleurs, nous mettrons beaucoup de lien vers des tutos et documentations externes afin de sourcer nos propos. Ici, tu ne trouveras pas de longues explications, mais deux formats :

Un descriptif Technique​

Chaque sujet sera traité via un Descriptif Technique qui sera volontairement court, mais qui proposera les bases afin de comprendre le sujet.

Un aide-mémoire​

Ce Descriptif Technique sera complété par une fiche Aide-mémoire. Elle sera claire et concise, tiendra sur une page et sera disponible en :

  • PDF,
  • et bien sûr Markdown

L'utilité ?​

Voici plusieurs cas d'usage :

  • support de travail pour ne pas à toujours faire des recherches
  • support de cours pour les profs
  • support de révision pour des examens ou préparer un entretien
  • support de préparation d'entretien professionel

Sourcer les infos​

Excepté dans les aide-mémoires, une profusion de liens te sera donner. Des liens Wikipédia afin que tu puisses mieux comprendre les sujets, les abréviations, les concepts, mais aussi des liens plus techniques ou pratiques.

Multi auteurs​

Afin de couvrir un maximum de sujet sur la vie des développeurs, d'autres personnes seront invités sur ce site. Mais ceci est une autre histoire.

Les auteurs​

Nathaniel Vaur Henel​

Créateur du site, je suis un dév spécialisé Java qui veut partager ses connaissances depuis longtemps. J'ai enfin trouvé une idée qui me plait ! Et j'espère qu'elle te plaira aussi.

Je suis dév senior, Tech lead (et toutes ses variations), formateur et maintenant auteur.

Je serais aussi le premier utilisateur de ce site !
En effet, j'en ai assez de faire des recherches pour retrouver un mot clef ou une comment faire, par exemple, on insère une image dans Markdown.

Les sujets traités ?​

N'importe quel sujet autour du développement pourra être traité.
Les premiers seront sur Java 17, le Markdown, les Regex (!).
Si tu as des idées, n'hésite à passer sur le github du site afin de poster une issue ou même pourquoi une PR !
